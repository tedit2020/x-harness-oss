## ネタ1: AI部署制

**元投稿:**  
> Claude Codeの中に「秘書室」「マーケ部」「エンジニア部」を作った。AIエージェントに部署ごとの役割を持たせると、1人開発なのに「引き継ぎ」が発生する。タスクの抜け漏れが明らかに減った。

**事実確認: ✅ 概ね正確（技術的根拠あり）**

**検証結果:**

- **部署制の実現可能性:** ✅ 確認済み。Claude CodeにはAgent Teams機能が存在し、複数のClaude Codeセッションをチームとして協調動作させることができる。各セッションに専用のsystem prompt・ツール権限・モデルを個別設定することで、「部署」に相当する役割分担が技術的に実現可能。
- **「引き継ぎ」の技術的実現方法:** ✅ 確認済み。Agent Teamsでの引き継ぎは、共有タスクリスト（TASKS.mdやJSONファイル等のプロジェクト内ファイル）を通じて行われる。各エージェントがファイルを読み書きし、タスクのステータス（pending/in-progress/done）を更新することで、直接メッセージ通信も含めた引き継ぎが発生する。サブエージェント単体ではセッション間通信ができないが、Agent Teamsはセッション間直接通信をサポート。
- **「タスクの抜け漏れが減った」:** ✅ 合理的な主張。タスク依存関係の自動管理機能があり、あるエージェントが完了したタスクに依存する他のタスクが自動でアンブロックされる仕組みが確認された。
- **注意:** Agent Teamsはデフォルト無効の実験的機能（`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` で有効化）。「実験的」である点は投稿に明示されていないが、事実として嘘ではない。

**投稿時の注意点:**
- Agent Teams機能は「実験的機能」であることを一言添えると誠実。
- 「部署を作った」という表現はやや比喩的だが、技術的裏付けがあるので問題なし。
- まっさんの実際の運用（クロコ組織）と合致しており、説得力のある実体験ネタとして使える。

## ネタ2: SaaSツール置き換え

**元投稿:**  
> ASO分析ツール（月3000円）→Claude Code + Python（月0円）、投稿自動化（月5000円）→自作スクリプト（月0円）、プロジェクト管理→Claude Codeのメモリで十分、日報作成→launchdで自動生成。API代込みでも月5000円以下

**事実確認: ⚠️ 一部要修正**

**検証結果:**

- **「月0円」の表記について:** ⚠️ 誇大表現の可能性あり。「Claude Code + Python（月0円）」「自作スクリプト（月0円）」は各ツール単体のコストを指していると解釈できるが、**Claude Code自体のコストが別途発生する**。Proプランは月$20（約3,000円）、Maxプランは月$100〜$200。この費用を除外して「月0円」と書くと誤解を招く。
- **「API代込みでも月5000円以下」について:** ⚠️ 条件付きで正確。API従量課金の場合、使用量によって変動する。Maxプランなら$100/月（約15,000円）固定のため「5000円以下」にならない。Proプランで軽量使用に留める場合は成立する可能性があるが、ヘビーユースには当てはまらない。ただし「API代込み」という文脈から、API従量課金ベースでの計算と解釈すれば、軽量使用なら月$30〜$40（5,000円前後）に収まるケースもあり、完全な嘘とは言えない。
- **launchdでの自動生成:** ✅ 技術的に正確。launchdはmacOSのジョブスケジューラであり、シェルスクリプト・Python・Node.jsなど任意のスクリプトを定期実行できる。日報生成スクリプトをlaunchdで定期起動することは標準的なmacOS自動化手法として確立されている。
- **プロジェクト管理→Claude Codeのメモリで十分:** ✅ 合理的な主張。ただし個人の用途・規模による。大規模チームには当てはまらない。

**投稿時の注意点:**
- ⚠️ 「月0円」の表記は修正推奨。「置き換え先ツール単体は0円（Claude Code本体費用は除く）」あるいは「ツール費用の削減額は月8,000円」のような正確な表現に変える。
- 「API代込みでも月5000円以下」はProプラン+軽量使用の前提を明示すれば成立する。
- 全体の趣旨（SaaSからClaude Codeへの乗り換えでコスト削減）は事実として支持できる。

## ネタ3: /ultraplan新機能

**元投稿:**  
> Claude Codeに新機能「/ultraplan」が登場。ClaudeがWeb上で実装プランを作成してくれる機能。ユーザーはプラン内容を確認、編集した上で、Web上またはターミナルに戻り実行することが可能。現在、Claude CodeのWeb版を使用できる全ユーザーにプレビュー版を提供中。

**事実確認: ✅ 正確**

**検証結果:**

- **機能の実在:** ✅ 確認済み。`/ultraplan`はClaude Codeの公式ドキュメント（code.claude.com/docs/en/ultraplan）に掲載された正式機能。
- **動作の仕組み:** ✅ 正確。ローカルCLIからクラウドのClaude Codeセッションへ計画タスクを引き渡し、クラウド側でOpus 4.6が最大30分かけて実装プランを作成。ユーザーはブラウザ上でインラインコメント・絵文字リアクション等で計画を確認・編集できる。実行はWeb上でもターミナルに戻っても可能。
- **「全ユーザーにプレビュー版を提供中」:** ✅ 正確。X上での公式告知（@trq212）でも「Available now in preview for all users with CC on the web enabled」と確認。ただし要件あり：Claude Code Web版（Pro/Max/Team/Enterprise）のアカウントとGitHubリポジトリの接続が必要。v2.1.91以降のClaude Codeが必要。
- **「Anthropicがリサーチプレビューと説明している」:** ✅ 正確。動作・仕様はフィードバックにより変更の可能性あり。
- **現時点の制限:** GitLabやBitbucket、ローカルのみのリポジトリは未対応。A/Bテストによりプランバリアント（シンプル/ビジュアル/詳細）を選択不可。

**投稿時の注意点:**
- 元投稿の内容はほぼ正確。「GitHubリポジトリの接続が必要」等の要件を一言添えると親切。
- 「プレビュー版」という表現は正確なので変更不要。

## ネタ4: Hooks自動テスト

**元投稿:**  
> Hooksを使って自動テスト環境を組めば、修正してテストしてエラーを直すループが完全に自動化される。

**事実確認: ⚠️ 概ね正確だが「完全に自動化」は要注意**

**検証結果:**

- **Hooks機能の実在:** ✅ 確認済み。Claude Code公式ドキュメント（code.claude.com/docs/en/hooks-guide）にて確認。2026年3月時点で21のライフサイクルイベント、4種のハンドラー型、非同期実行、JSON構造化出力に対応した本格的な自動化プラットフォームへ成長している。
- **Hooksでのテスト自動実行:** ✅ 技術的に正確。PostToolUseフックで `"write:src/**/*.ts"` のようなマッチャーを使い、ソースファイル変更後に `npm test -- --related` を自動実行する設定が可能。Stopフックで「テストを実行して合格するまでClaudeを停止させない」構成も実現できる。
- **「完全に自動化される」の表現:** ⚠️ やや誇大。以下の制限がある：
  1. **無限ループリスク:** StopフックがExit 2を返すとClaudeが作業継続するが、`stop_hook_active` フラグのチェックを怠ると無限ループが発生する。
  2. **技術的難易度:** 設定・維持に深い技術知識が必要。
  3. **セキュリティ:** フックは任意コマンドを実行するため、管理・セキュリティのオーバーヘッドが存在する。
  4. **「自分でテストを直す」部分:** AIが自律的にエラーを修正する能力はコードの複雑さや問題の性質に依存し、常に成功するとは限らない。
- **「DXを根本的に変える」:** ✅ 主観的表現だが誇大ではない。開発ループの自動化は実際に有意な効率改善をもたらすことが複数の事例で確認されている。

**投稿時の注意点:**
- ⚠️ 「完全に自動化される」→「大幅に自動化できる」「ほぼ自動化できる」に修正推奨。完全自動化にはエッジケースや設定の落とし穴がある。
- Hooksの設定には技術知識が必要な点を一言添えると誠実。
- 全体の主旨（Hooksでテスト自動化ループが組める）は事実として支持できる。
